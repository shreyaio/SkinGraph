# SkinGraph — Graph, Scoring, and Risk Matrix (Conceptual Overview)

## 1. What is the Graph?

The SkinGraph system is built on a **graph-based knowledge structure**.

A graph consists of:

* **Nodes** → Ingredients (e.g., Retinol, Lactic Acid)
* **Edges** → Relationships between ingredients (e.g., conflict, synergy)

### Example:

Retinol ──(conflict: 0.72)──► Lactic Acid
Retinol ──(synergy: 0.40)──► Niacinamide

Each edge stores structured scientific information:

* relation_type (CONFLICT / SYNERGY / etc.)
* base_score (0–1)
* mechanism (e.g., barrier disruption)
* source_count, agreement_ratio
* pih_risk, Fitzpatrick coverage

👉 The graph represents **population-level scientific knowledge**.

---

## 2. Schema (Data Contracts)

All data in the system follows strict dataclass structures:

* `NodeData` → describes an ingredient
* `EdgeData` → describes interaction between two ingredients
* `SkinProfile` → user-specific attributes
* `ConfidenceReport` → reliability of the interaction
* `RiskMatrix` → full routine-level evaluation

👉 After this stage, **no raw dictionaries are used** — everything is typed and structured.

---

## 3. Scoring System

Scoring converts relationships into **quantified risk values (0–1)**.

### 3.1 Base Score (Graph)

From the graph:

```
Retinol + Lactic Acid → base_score = 0.72
```

This represents population-level risk.

---

### 3.2 Skin-Adjusted Score (Personalisation)

The score is adjusted using the user's skin profile:

Example:

```
SkinProfile:
- Fitzpatrick: IV
- Sensitivity: High
- Concern: Hyperpigmentation
```

Adjustments:

* PIH risk + darker skin tone → increase
* High sensitivity → increase

```
0.72 × 1.15 × 1.25 ≈ 1.035 → capped to 1.0
```

👉 This becomes:

```
skin_adjusted_score = 1.0
```

---

### 3.3 ML Feedback Score

A LightGBM model refines the estimate using learned patterns.

Input features include:

* base_score
* skin_adjusted_score
* source_count
* agreement_ratio
* skin profile flags
* ingredient irritancy
* PIH risk

Example output:

```
ml_score = 0.81
```

👉 ML corrects:

* bias in literature
* underrepresentation of certain skin types
* real-world formulation effects

---

### 3.4 Final Score (Ensemble)

```
final_score =
  0.5 * skin_adjusted_score +
  0.3 * ml_score +
  0.2 * base_score
```

Example:

```
= 0.5 * 1.0 + 0.3 * 0.81 + 0.2 * 0.72
= 0.887 ≈ 0.89
```

👉 Final interpretation:

* 0.0–0.3 → Safe
* 0.3–0.6 → Caution
* 0.6–1.0 → High Risk

---

## 4. Confidence Layer

Each interaction includes a confidence score:

```
source_confidence = min(source_count / 10, 1.0)
agreement_confidence = agreement_ratio
coverage_confidence = 1.0 or 0.75 (if limited Fitzpatrick data)

overall_confidence =
  0.4 * source_confidence +
  0.4 * agreement_confidence +
  0.2 * coverage_confidence
```

Example:

```
Confidence ≈ 0.88 (High)
```

👉 Adds transparency:

* highlights uncertainty
* flags lack of data for darker skin tones

---

## 5. Risk Matrix (Routine-Level Risk)

When multiple ingredients are used together, we evaluate **all pairwise interactions**.

### 5.1 Generate all pairs

For ingredients:

```
A = Retinol
B = Lactic Acid
C = Vitamin C
```

Pairs:

```
(A, B), (A, C), (B, C)
```

---

### 5.2 Compute scores for each pair

```
(A, B) → 0.89
(A, C) → 0.55
(B, C) → 0.65
```

---

### 5.3 Compute cumulative risk

Instead of average or max, we use:

```
cumulative_risk = 1 - ∏(1 - score_i)
```

Example:

```
= 1 - (1-0.89)(1-0.55)(1-0.65)
= 1 - (0.11 × 0.45 × 0.35)
= 1 - 0.0173
= 0.9827 ≈ 0.98
```

---

### 5.4 Interpretation

Even if only one pair is highly risky:

👉 Multiple moderate conflicts combine to create **very high overall stress**

This models real-world skin behavior:

* irritation is cumulative
* barrier damage compounds across products

---

## 6. RiskMatrix Object

The system stores:

```
RiskMatrix(
  edges=[EdgeData(...), EdgeData(...), EdgeData(...)],
  cumulative_burden_score=0.98
)
```

---

## 7. Key Insight

* Graph → What interacts
* Scoring → How risky it is
* ML → Corrects the estimate
* RiskMatrix → How everything interacts together
* Confidence → How much we trust it

---

## 8. Final Summary

SkinGraph is not a rule-based system.

It is a **multi-layer reasoning system**:

1. Graph encodes scientific relationships
2. Schema enforces structure
3. Scoring personalises risk
4. ML refines predictions
5. RiskMatrix evaluates full routine impact

👉 Result:
A personalised, explainable, and scalable skincare intelligence engine.
